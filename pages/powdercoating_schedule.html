Powdercoating never knew in advance what colour powder and how much they would need. A lot of jobs were being help up because they didn't have the colour in stock or not enough powder to cover the job.

üôå Solution:

An easy to read schedule which they can quickly check to see what jobs are coming up and what colours need to be ordered.

The powdercoating schedule was added to a Laravel project I previously made that used our ordering softwares API's to generate lists for despatch.

A python program runs once a day calling the API to generate a list of jobs that are in the schedule. It loops over each jobs checking each part if it is to be powdercoated. If the ticket number doesn't already exist in my database it gets added recording the schedule date, ticket number, customer, part code, what colour and how much powder the part will need.

Home page. The Powdercoating Calendar was added to a Laravel project I previously made.

The calendar view (red == powder has not been checked; green == powder has already been checked)

Clicking on one of the items in the calendar will open a modal showing all the information that is relevant to that part. The data for the modal is fetched using AJAX. Clicking on 'Powder Checked' sends off another AJAX request using Laravel to update to update the database, when it gets a successful response. The corresponding item in the calendar get changed from red (not checked) to green (checked).

By clicking on one of the days at the top of calender, this opens the day view. Where the user will only see the parts that are scheduled for that day. This page displays the data using DataTables which has a great filter box built in, great for filtering results for a certain colour. I added a bit of javascript to the page so the user could select multiple items and the page will display the total meters squared.
